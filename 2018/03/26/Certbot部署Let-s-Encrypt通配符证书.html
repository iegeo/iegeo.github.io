Certbot部署Let's Encrypt通配符证书

发表于 2018-03-26 | 分类于 资料

要在网站上启用HTTPS，需要从证书颁发机构（CA）获取证书。Let’s Encrypt是由非营利互联网安全研究组织（ISRG）提供的免费，自动化和开放的认证机构，目前已支持通配符SSL证书。我们可以从Let’s Encrypt获得网站域名的证书。官网目前推荐的获取和安装方式是 certbot ，只需要简单运行一些命令并作一些配置即可。

本文以ubuntu server 16.04系统为例。

安装certbot

为了实现通配符证书，Let’s Encrypt 对 ACME 协议的实现进行了升级，只有 v2 协议才能支持通配符证书。Certbot 0.22.0 以上版本支持新ACME v2 协议。

  1. 官方源安装

    apt update
apt install software-properties-common
add-apt-repository ppa:certbot/certbot
apt update
apt install certbot

  2. 脚本安装（需要512MB以上的RAM）如果内存小于512MB，可以先设置临时交换文件

    fallocate -l 1G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
wget https://dl.eff.org/certbot-auto
chmod a+x ./certbot-auto
./certbot-auto --help #查看具体情况
./certbot-auto --**

Certbot插件

客户在申请 Let’s Encrypt 证书的时候，需要校验域名的所有权，证明操作者有权利为该域名申请证书，目前支持三种验证方式：

  • dns-01：给域名添加一个 DNS TXT 记录。
  • http-01：在域名对应的 Web 服务器下放置一个 HTTP well-known URL 资源文件。
  • tls-sni-01：在域名对应的 Web 服务器下放置一个 HTTPS well-known URL 资源文件。

而申请通配符证书，只能使用 dns-01 的方式。如果要实现证书自动续订，则需要配合dns插件来调用域名解析商的api来实现自动 dns-01 验证。
官方提供的dns插件

安装插件

pip install certbot-dns-cloudflare #或者其他
certbot plugins #查看已安装的插件

可以看到

root@ubuntu:~# certbot plugins
Saving debug log to /var/log/letsencrypt/letsencrypt.log

* dns-cloudflare
Description: Obtain certificates using a DNS TXT record (if you are using
Cloudflare for DNS).
Interfaces: IAuthenticator, IPlugin
Entry point: dns-cloudflare =
certbot_dns_cloudflare.dns_cloudflare:Authenticator

* standalone
Description: Spin up a temporary webserver
Interfaces: IAuthenticator, IPlugin
Entry point: standalone = certbot.plugins.standalone:Authenticator

* webroot
Description: Place files in webroot directory
Interfaces: IAuthenticator, IPlugin
Entry point: webroot = certbot.plugins.webroot:Authenticator

获取证书

创建dns-cloudflare配置文件

touch /etc/letsencrypt/cloudflare.ini
vim /etc/letsencrypt/cloudflare.ini
#示例如下
dns_cloudflare_email = cloudflare@example.com 
dns_cloudflare_api_key = 0123456789abcdef0123456789abcdef01234567

产生证书

certbot certonly \
   --server "https://acme-v02.api.letsencrypt.org/directory" \
   --dns-cloudflare \
   --dns-cloudflare-credentials /etc/letsencrypt/cloudflare.ini \
   -d yuming.com -d *.yuming.com

等待命令执行，根据提示输入，如果一切正常就可以获得证书了。
